# 시작하기 (Getting Started)

## 카테고리 소개

Ouroboros를 처음 사용하시는 분들을 위한 카테고리입니다. 설치부터 환경 확인, 기초 학습까지 Ouroboros 워크플로우에 발을 들이기 위한 모든 준비 과정을 이 카테고리 하나로 완성하실 수 있습니다. 순서대로 따라가시면 10분 안에 Ouroboros를 완전히 사용할 준비가 됩니다.

이 카테고리의 스킬들은 별도의 사전 지식 없이도 누구나 실행할 수 있도록 설계되어 있습니다. Python과 Claude Code가 설치된 환경이라면 곧바로 시작하실 수 있으며, MCP 서버가 없어도 핵심 기능 대부분을 폴백(fallback) 모드로 체험하실 수 있습니다.

Ouroboros의 핵심 철학은 "코드를 짜기 전에, 무엇을 만들지를 먼저 정의한다"는 것입니다. 시작하기 카테고리는 그 여정의 출발점입니다. 환경을 설정하고, 상태를 확인하고, 튜토리얼을 통해 전체 사이클(인터뷰 → 시드 → 실행 → 평가)을 직접 체험해 보세요.

언제든지 막히는 부분이 있으면 `ooo help` 명령어로 전체 명령어 목록을 조회하거나, `ooo welcome`으로 현재 시스템 상태와 권장 다음 단계를 확인하실 수 있습니다. Ouroboros는 항상 사용자가 다음에 무엇을 해야 할지 안내해 드립니다.

## 이 카테고리의 스킬 한눈에 보기

| 스킬명 | 한 줄 설명 | 난이도 | 예상 소요 시간 |
|--------|-----------|--------|---------------|
| setup (설치) | MCP 서버 등록과 환경 설정을 6단계 마법사로 완성 | 입문 | 5분 |
| welcome (환영) | 현재 시스템 상태와 권장 다음 행동을 한눈에 확인 | 입문 | 2분 |
| tutorial (튜토리얼) | 전체 Ouroboros 사이클을 핸즈온으로 체험 | 입문 | 10분 |
| help (도움말) | 전체 명령어 목록과 예시를 즉시 조회 | 입문 | 1분 |
| update (업데이트) | PyPI 최신 버전 확인 및 업그레이드 안내 | 입문 | 2분 |
| cancel (실행 중단) | 실행 중인 작업을 안전하게 취소하고 정리 | 입문 | 1분 |

## 추천 실행 순서

![Diagram 1](../assets/diagrams/categories__getting-started__diagram_1.svg)

1. **`ooo setup`** — 가장 먼저 실행하세요. MCP 서버를 등록하고 환경을 검증합니다.
2. **`ooo welcome`** — 설치 후 현재 상태를 확인하고 권장 다음 단계를 파악합니다.
3. **`ooo tutorial`** — 인터뷰 → 시드 → 실행 → 평가 전 과정을 직접 체험합니다.
4. **`ooo help`** — 언제든지 전체 명령어 목록이 필요할 때 호출합니다.
5. **`ooo update`** — 주기적으로 최신 버전을 유지하세요.
6. **`ooo cancel`** — 실행 중인 작업을 중단해야 할 때 사용합니다.

## 관련 카테고리

| 카테고리 | 연결 지점 |
|---------|----------|
| 요구사항 정의 (Requirements) | `setup` 완료 후 `ooo interview` → `ooo seed`로 진행 |
| 개발 실행 (Execution) | `tutorial`에서 `ooo run`을 체험한 뒤 본격 실행 단계로 이동 |

## 스킬 상세

### 1. 설치 (setup)

> **난이도**: 입문 | **소요 시간**: 5분 | **명령어**: `ooo setup` 또는 `/ouroboros:setup`

> 소스: skills/setup/SKILL.md

Ouroboros 사용에 필요한 모든 환경을 6단계 마법사 형식으로 설정합니다. MCP 서버 등록, CLAUDE.md 통합, 브라운필드 스캔까지 한 번에 완료할 수 있습니다.

#### 실행 전 체크리스트

- [ ] Python 3.12 이상이 설치되어 있는지 확인
- [ ] Claude Code가 설치되어 있는지 확인 (`claude --version`)
- [ ] 인터넷 연결 상태 확인 (PyPI 패키지 다운로드 필요)

#### Step-by-Step 실행 가이드

**Step 1: 명령어 실행**

```
ooo setup
```

또는 Claude Code 슬래시 명령어로:

```
/ouroboros:setup
```

**Step 2: 마법사 단계별 안내**

| 단계 | 내용 | 사용자 응답 |
|------|------|------------|
| Phase 0: 환영 | Ouroboros 소개 및 GitHub Star 안내 | 계속 진행 여부 확인 |
| Phase 1: 환경 감지 | Python 버전, Claude Code 설치 여부 자동 감지 | — |
| Phase 2: MCP 등록 | MCP 서버를 `mcp.json`에 자동 등록 | 경로 확인 |
| Phase 3: CLAUDE.md 통합 | 프로젝트 CLAUDE.md에 Ouroboros 섹션 추가 | 통합 여부 선택 |
| Phase 4: 검증 | `ooo welcome`을 통해 설치 결과 확인 | — |
| Phase 5: 브라운필드 스캔 | 기존 프로젝트 구조 분석 (선택 사항) | 스캔 여부 선택 |

**Step 3: 산출물 확인**

설치 완료 후 터미널에 다음과 같은 확인 메시지가 출력됩니다.

#### MCP 없이 사용하기

MCP 서버 없이도 수동으로 설치할 수 있습니다. 터미널에서 다음 명령어를 실행하세요.

```bash
pip install ouroboros-ai[claude]
```

이후 프로젝트의 `.claude/mcp.json` 파일에 아래 항목을 직접 추가합니다.

```json
{
  "mcpServers": {
    "ouroboros": {
      "command": "python",
      "args": ["-m", "ouroboros.mcp"]
    }
  }
}
```

#### 산출물 실제 예시

```
✅ Ouroboros Setup Complete

Environment:
  Python: 3.12.4 ✓
  Claude Code: 1.x.x ✓
  MCP Server: registered ✓

Next recommended action:
  → Run `ooo welcome` to verify your environment
```

### 2. 환영 (welcome)

> **난이도**: 입문 | **소요 시간**: 2분 | **명령어**: `ooo welcome` 또는 `/ouroboros:welcome` (또는 그냥 `ooo`)

> 소스: skills/welcome/SKILL.md

현재 시스템 상태(페르소나 감지, MCP 연결 상태)를 즉시 확인하고, 상황에 맞는 권장 다음 행동을 안내합니다. Ouroboros 사용 중 언제든지 현재 위치를 파악하고 싶을 때 사용하세요.

#### 실행 전 체크리스트

- [ ] `ooo setup`이 완료된 상태인지 확인 (처음 사용 시)
- [ ] Claude Code 세션이 활성화되어 있는지 확인

#### Step-by-Step 실행 가이드

**Step 1: 명령어 실행**

```
ooo
```

또는 명시적으로:

```
ooo welcome
```

**Step 2: 출력 내용 이해**

| 항목 | 설명 |
|------|------|
| 페르소나 감지 | 현재 세션에서 활성화된 Ouroboros 페르소나 |
| MCP 연결 상태 | MCP 서버 정상 동작 여부 |
| 빠른 참조 카드 | 현재 상황에서 쓸 수 있는 명령어 목록 |
| 권장 다음 행동 | 세션 상태 기반으로 제안하는 다음 단계 |

**Step 3: 권장 행동 따르기**

출력 하단의 "권장 다음 행동" 섹션을 확인하고 해당 명령어를 실행합니다.

#### MCP 없이 사용하기

MCP 서버가 없어도 기본 명령어 참조 카드를 표시합니다. MCP 연결 상태는 "미연결"로 표시되며, MCP 기반 기능(세션 상태 조회 등)은 사용할 수 없습니다.

#### 산출물 실제 예시

```
👋 Ouroboros에 오신 것을 환영합니다!

시스템 상태:
  MCP 서버: ✅ 연결됨
  활성 세션: 없음

빠른 명령어 참조:
  ooo interview   요구사항 인터뷰 시작
  ooo seed        시드 문서 생성
  ooo run         실행
  ooo evaluate    평가
  ooo tutorial    핸즈온 튜토리얼

권장 다음 행동:
  → 새 프로젝트를 시작하려면 `ooo interview`를 실행하세요
```

### 3. 튜토리얼 (tutorial)

> **난이도**: 입문 | **소요 시간**: 10분 | **명령어**: `ooo tutorial` 또는 `/ouroboros:tutorial`

> 소스: skills/tutorial/SKILL.md

Ouroboros의 전체 워크플로우(인터뷰 → 시드 → 실행 → 평가)를 실제 예제를 통해 단계별로 체험합니다. 처음 사용자가 핵심 개념과 사용법을 빠르게 익힐 수 있도록 설계된 핸즈온 가이드입니다.

#### 실행 전 체크리스트

- [ ] `ooo setup` 및 `ooo welcome` 완료 상태
- [ ] 약 10분의 여유 시간 확보
- [ ] 간단한 프로젝트 아이디어 하나 준비 (없어도 예제 제공)

#### Step-by-Step 실행 가이드

**Step 1: 명령어 실행**

```
ooo tutorial
```

**Step 2: 5단계 핸즈온 진행**

| 단계 | 내용 |
|------|------|
| Phase 0: 컨텍스트 설정 | 튜토리얼 목표와 진행 방식 안내 |
| Phase 1: 인터뷰 체험 | 소크라테스식 질문으로 요구사항 명확화 과정 체험 |
| Phase 2: 시드 생성 체험 | 인터뷰 결과로 시드 문서 생성 과정 체험 |
| Phase 3: 실행 체험 | 시드 문서를 바탕으로 실행 단계 체험 |
| Phase 4: 평가 체험 | 실행 결과를 평가하고 피드백 루프 이해 |

**Step 3: 산출물 검토**

각 단계 완료 후 생성된 산출물(시드 문서, 실행 계획, 평가 보고서)을 확인하고 실제 프로젝트에서 어떻게 활용할지 파악합니다.

#### MCP 없이 사용하기

MCP 서버 없이도 튜토리얼의 핵심 내용을 체험할 수 있습니다. MCP 없이는 인터뷰와 시드 생성 단계는 정상 동작하나, 실행(`ooo run`) 단계는 MCP 없이 진행됩니다. 이 경우 실행 단계는 가이드 형식의 안내로 대체됩니다.

#### 산출물 실제 예시

```markdown
# 튜토리얼 완료 보고서

## 체험한 워크플로우
- ✅ Phase 1: 인터뷰 (소크라테스식 질의응답 5회)
- ✅ Phase 2: 시드 문서 생성
- ✅ Phase 3: 실행 체험
- ✅ Phase 4: 평가 및 피드백

## 생성된 시드 문서 미리보기
목표: 할 일 관리 앱
핵심 기능: 태스크 생성, 우선순위 설정, 완료 처리
...

## 다음 단계 권장
→ 실제 프로젝트에 `ooo interview`를 적용해 보세요
```

#### 다음 단계

- 요구사항 정의 카테고리의 [interview (인터뷰)] — 실제 프로젝트 인터뷰 시작
- 요구사항 정의 카테고리의 [seed (시드)] — 실제 시드 문서 생성

### 4. 도움말 (help)

> **난이도**: 입문 | **소요 시간**: 1분 | **명령어**: `ooo help` 또는 `/ouroboros:help`

> 소스: skills/help/SKILL.md

Ouroboros의 모든 명령어 목록, 각 명령어의 사용 방법, 예시를 즉시 조회합니다. 작업 중 명령어가 기억나지 않을 때 언제든지 호출할 수 있습니다.

#### 실행 전 체크리스트

- [ ] 별도 사전 조건 없음 (MCP 없이도 동작)

#### Step-by-Step 실행 가이드

**Step 1: 명령어 실행**

```
ooo help
```

**Step 2: 출력 내용 확인**

| 섹션 | 내용 |
|------|------|
| 전체 명령어 표 | 모든 `ooo` 명령어와 한 줄 설명 |
| 빠른 시작 가이드 | 처음 시작하는 사람을 위한 순서 안내 |
| 예시 | 각 명령어의 실제 사용 예 |

**Step 3: 필요한 명령어 확인 후 실행**

도움말에서 원하는 명령어를 찾아 바로 실행합니다.

#### MCP 없이 사용하기

`ooo help`는 MCP 서버 없이도 완전히 동작합니다. 전체 명령어 목록과 설명은 항상 제공됩니다. 단, MCP가 필요한 개별 명령어의 고급 기능은 MCP 연결 시에만 사용 가능합니다.

#### 산출물 실제 예시

```
Ouroboros 명령어 목록

시작하기:
  ooo setup       환경 설정 및 MCP 등록
  ooo welcome     현재 상태 확인
  ooo tutorial    핸즈온 튜토리얼

개발 워크플로우:
  ooo interview   소크라테스식 요구사항 인터뷰
  ooo seed        시드 문서 생성
  ooo run         실행
  ooo evaluate    평가
  ooo evolve      진화 단계 실행

유틸리티:
  ooo status      세션 및 드리프트 상태 확인
  ooo unstuck     막혔을 때 창의적 돌파구 탐색
  ooo qa          품질 검증
  ooo update      버전 업데이트 확인
  ooo cancel      실행 중단
  ooo help        이 도움말 보기
```

#### 다음 단계

- [welcome (환영)](#2-환영-welcome) — 현재 상태 확인

### 5. 업데이트 (update)

> **난이도**: 입문 | **소요 시간**: 2분 | **명령어**: `ooo update` 또는 `/ouroboros:update`

> 소스: skills/update/SKILL.md

PyPI에서 Ouroboros 최신 버전을 확인하고 업그레이드 명령어를 안내합니다. 현재 설치된 버전과 최신 버전을 비교하여 업데이트가 필요한 경우 안내합니다.

#### 실행 전 체크리스트

- [ ] Ouroboros가 이미 설치되어 있는지 확인
- [ ] 인터넷 연결 상태 확인 (PyPI 조회 필요)

#### Step-by-Step 실행 가이드

**Step 1: 명령어 실행**

```
ooo update
```

**Step 2: 버전 비교 확인**

| 항목 | 설명 |
|------|------|
| 현재 설치 버전 | 현재 환경에 설치된 Ouroboros 버전 |
| PyPI 최신 버전 | 현재 배포된 최신 버전 |
| 업데이트 필요 여부 | 버전 비교 결과 |

**Step 3: 업그레이드 실행 (필요 시)**

안내된 업그레이드 명령어를 그대로 터미널에 실행합니다.

```bash
pip install --upgrade ouroboros-ai[claude]
```

#### MCP 없이 사용하기

MCP 서버 없이도 버전 확인 기능은 정상 동작합니다. 수동으로 업그레이드할 경우 아래 명령어를 직접 실행하세요.

```bash
pip install --upgrade ouroboros-ai[claude]
```

업그레이드 후에는 `ooo welcome`으로 새 버전이 정상 동작하는지 확인하는 것을 권장합니다.

#### 산출물 실제 예시

```
Ouroboros 버전 확인

현재 설치 버전: 0.9.1
PyPI 최신 버전:  0.9.3

⬆️  업데이트가 있습니다!

업그레이드 명령어:
  pip install --upgrade ouroboros-ai[claude]

업그레이드 후 확인:
  ooo welcome
```

### 6. 실행 중단 (cancel)

> **난이도**: 입문 | **소요 시간**: 1분 | **명령어**: `ooo cancel` 또는 `/ouroboros:cancel`

> 소스: skills/cancel/SKILL.md

현재 실행 중인 Ouroboros 작업을 안전하게 취소하고, 남아 있는 고아 작업(orphaned jobs)을 정리합니다. 실행이 예상보다 오래 걸리거나 잘못된 방향으로 진행될 때 사용합니다.

#### 실행 전 체크리스트

- [ ] 취소하려는 실행 중인 작업이 있는지 확인 (`ooo status`로 확인 가능)
- [ ] 취소 시 현재 진행 중인 작업의 중간 결과가 손실될 수 있음을 인지

#### Step-by-Step 실행 가이드

**Step 1: 명령어 실행**

```
ooo cancel
```

**Step 2: 취소 대상 확인**

| 항목 | 설명 |
|------|------|
| 실행 중인 작업 목록 | 현재 활성화된 모든 작업 표시 |
| 취소 확인 | 취소할 작업 선택 또는 전체 취소 |

**Step 3: 취소 완료 확인**

취소 확인 메시지를 확인하고, 필요 시 `ooo welcome`으로 현재 상태를 재확인합니다.

#### MCP 없이 사용하기

MCP 서버가 없는 경우, 현재 활성화된 프로세스 목록을 출력하고 수동으로 종료할 수 있도록 안내합니다. 프로세스 PID를 확인하여 직접 `kill` 명령어로 종료하는 방법을 제시합니다.

```bash
# 수동 종료 예시
kill <PID>
```

#### 산출물 실제 예시

```
실행 중인 작업:

  [1] job-abc123   ooo run   시작: 14:32:05   상태: 진행중
  [2] job-def456   ooo evolve  시작: 14:35:10   상태: 대기중

취소할 작업 번호를 입력하세요 (전체: all, 취소 없이 종료: q):
> all

✅ 모든 작업이 취소되었습니다.
  job-abc123: 취소됨
  job-def456: 취소됨
```
